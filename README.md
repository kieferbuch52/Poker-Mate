# Poker Mate v6.0.0

スマホ向けのポーカー収支・学習PWAです。

## 主な機能

- リング／トーナメントの収支管理
- 累積収支グラフ、リング時給、MTT ROI
- 店舗別成績、チップ残高、通貨・手動換算レート
- 店舗でのチップ購入・換金履歴
- リング／トーナメントのRFIレンジ
- 相手のオープン位置別BBディフェンス
- パワーナンバー表とM値・必要PN計算
- ベットとレイズに対応したオッズ計算
- 自分／相手／ボード指定のヘッズアップ勝率完全計算
- アウツ数から次の1枚・リバーまでのドロー勝率計算
- バンクロール残高、入出金、リングBI数、MTT参加可能回数
- ハンドメモ
- JSONバックアップ、CSV出力
- オフライン利用

## v6.0.0の更新内容

### セッション計測

- 開始・一時停止・再開・終了に対応したタイマーを追加
- タイマーはブラウザを閉じても端末内に保持
- 開始時刻と終了時刻を保存
- 日をまたぐセッションのプレー時間を自動計算
- 時刻を手入力した場合もプレー時間へ反映

### 経費内訳

経費を次の5種類に分けて保存します。

- 交通
- 飲食
- 宿泊
- チップ
- その他

旧バージョンの経費は、データ移行時に「その他」として引き継ぎます。

### 成績分析

- ステークス別の回数・時間・勝率・時給・収支
- 店舗別の勝率・平均時間・最大勝ち・最大負け・直近10回
- MTTのITM率・FT率・優勝回数・ABI・リエントリー率・平均参加人数
- 現在のドローダウン
- 最大ドローダウン
- 月次・年次レポート
- リエントリー回数の記録

### データ形式

- 保存データをスキーマv6へ更新
- 過去データを自動移行
- CSVへ開始・終了時刻、経費内訳、リエントリー回数を追加

## v4.3.0の更新内容

### 設定画面の整理

- リングステークス候補と基本設定を別パネルへ分離
- リングステークス候補を設定画面の上部へ配置
- 基準通貨とセッション損失上限を基本設定パネルへ移動
- 基本設定パネルをリングステークス候補より下へ配置
- アプリバージョン表示を基本設定パネル内へ移動

### 数値入力の不具合修正

数値候補を選択した際、整数末尾のゼロが消える不具合を修正しました。

修正前：

- `100` → `1`
- `500` → `5`
- `1,000` → `1`

修正後：

- `100` → `100`
- `500` → `500`
- `1,000` → `1,000`
- `1.50` → `1.5`

### 計算ツールの再編

計算ツールを次の4種類へ整理しました。

1. オッズ計算
2. 勝率計算
3. アウツ
4. パワーナンバー

- 4つのツールをスマホ画面内の1行に表示
- ツールアイコンとラベルをコンパクト化
- 従来の「ベット対応」と「対レイズ」を「オッズ計算」へ統合
- オッズ計算内で「ベットにコール」と「レイズにコール」を切り替え可能
- 「アウト」の表記を「アウツ」へ統一
- 過去のバージョンで対レイズを開いていた場合も、更新後はオッズ計算へ自動移行

## v5.0.0の更新内容

### データ信頼性

- 日付をUTCではなく端末の現地日付で保存するよう修正
- データスキーマをv5へ更新し、旧データを自動移行
- 店舗チップ残高を「導入時残高＋購入・換金＋セッション増減」から毎回計算
- インポート前の構造検証、ID重複、日付、店舗参照、数値範囲の確認を追加
- 設定画面にデータ整合性診断を追加

### セッションと店舗チップ

リングセッションでは、店舗チップの扱いを次から選択できます。

1. 保有チップから参加
2. 現金で直接参加
3. 今回チップを購入して参加

「今回チップを購入して参加」では購入台帳も自動作成されるため、同じ購入の二重入力を防げます。

### バンクロール定義

総ポーカー資産は次の式で計算します。

`開始資金＋外部入出金＋セッション損益`

店舗チップは総資産の内訳であり、総資産へ別途加算しません。画面では総資産、店舗チップ、手元現金等の推定額を分けて表示します。

### バックアップ

- 端末内へ最大5件の自動スナップショットを保存
- インポート・全削除・復元前は強制的に退避
- 最新の自動バックアップから復元可能
- 最終JSON書き出しから14日以上経過した場合に警告

### オッズ計算

- 3つの入力値を1行に集約
- レイズにコールの計算ボタンを廃止し、入力と同時に再計算
- ベットサイズ表を廃止し、両モード共通の「レイズサイズ別 必要勝率表」へ統一

### 自動テスト

`npm test` または `node tests/run-tests.js` で、日付、オッズ、チップ台帳、全ポーカー役の強弱、キッカー、ボードプレイを確認できます。GitHubへpush／Pull Requestすると、GitHub Actionsでも自動実行されます。

## 完全勝率計算について

- モンテカルロ法や乱数は使用せず、残りカードの全組み合わせを列挙します。
- 相手のハンドを指定した場合は、プリフロップを含めて完全計算できます。
- 相手がランダムの場合は、フロップ・ターン・リバーから完全計算できます。
- プリフロップ対ランダムは2,097,572,400通りになるため、スマホでは相手の2枚を指定する仕様です。
- 重い計算はWeb Workerで実行し、進捗表示と計算中止に対応しています。

## GitHub Pagesで公開

1. 新しいGitHubリポジトリを作成
2. ZIPを解凍し、フォルダ内のファイルをすべてリポジトリ直下へアップロード
3. `Settings` → `Pages` を開く
4. Sourceを「Deploy from a branch」にする
5. Branchを `main`、Folderを `/ (root)` にして保存
6. 公開URLをスマホのChromeで開き、「ホーム画面に追加」

既にGitHub Pagesへ公開している場合は、ZIP内のファイルをすべて上書きしてください。

## 注意

- データは端末のブラウザ内（localStorage）に保存されます。
- 機種変更やブラウザデータ削除に備え、定期的にJSONバックアップを保存してください。
- 同じGitHub PagesのURLへ上書き更新すれば、既存データは引き継がれます。
- 換算レートは自動更新ではなく手動入力です。
- レンジとパワーナンバーは学習用の目安です。
- ICM、レーキ、相手傾向、スタック深度に応じて調整してください。
