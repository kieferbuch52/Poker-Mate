# Poker Mate v8.1.1

スマホ向けのポーカー収支・学習PWAです。

## 主な機能

- リング／トーナメントの収支管理
- 累積収支グラフ、リング時給、MTT ROI
- 店舗別成績、チップ残高、通貨・手動換算レート
- 店舗でのチップ購入・換金履歴
- リング／トーナメントのRFIレンジ
- 相手のオープン位置別BBディフェンス
- パワーナンバー表とM値・必要PN計算
- ベットとレイズに対応したオッズ計算
- 自分／相手／ボード指定のヘッズアップ勝率完全計算
- アウツ数から次の1枚・リバーまでのドロー勝率計算
- バンクロール残高、入出金、リングBI数、MTT参加可能回数
- ハンドメモ
- JSONバックアップ、CSV出力
- オフライン利用

## v8.1.0の更新内容

### ヨコサワ式：リング／トーナメント

- リング：100BB・アンティなし
- トーナメント：アンティあり・40BB以上の簡易応用

トーナメントでは、リングの参加基準を1ランク広げます。25BB以下や強いICM局面ではPoker Mateのトーナメントレンジを使用するよう案内します。

### ヨコサワ式：3BETされた場面

「First in」と「3BETされた」を切り替えられます。3BETされた場面では、各ハンドを4BET・CALL・FOLDで表示します。

簡易ルール：

1. 自分のオープン下限より2ランク強い色を相手の3BET下限と推定
2. 相手の下限と同じ色、または1ランク強い色はCALL
3. 相手の下限より2ランク以上強い色は4BET
4. 相手の推定下限より弱い色はFOLD

3BETサイズ、ポジション、レーキ、ICMを省略した初心者向け判断です。

## v7.0.0の更新内容

### レンジの前提と頻度

- 卓人数を6-max／9-max簡易補正から選択
- トーナメントのアンティ有無とchipEV／バブル／FTを選択
- BBディフェンスで相手の2.0〜4.0BBオープンサイズを選択
- 基準データ、現在の前提、簡易補正内容を画面内へ表示
- ハンドごとに0・25・50・75・100%の頻度を登録可能
- 頻度は端末内に保存し、条件別に管理
- 既定レンジがソルバーの生データではないことを明記
- トーナメント15BBを「2BBオープン／オールイン混合」、10BBを「オールイン候補」と明示

### 勝率計算

- Player 1とPlayer 2に簡易上位10・20・30・50%レンジを指定可能
- 指定ハンド対レンジ、レンジ対レンジをフロップ以降で完全列挙
- Player 3を追加し、3人ポットを完全列挙
- 3人ポットは全員のハンド指定時に対応
- 計算前に組み合わせ数の概算を表示
- 正確な計算が2,000万通りを超える場合は条件変更を案内
- 「完全計算」から「完全列挙・条件あり」へ表記を修正

### オッズ・アウツ・パワーナンバー

- オッズ計算へレーキ率と上限を追加
- レーキ控除後のポットで必要勝率を計算
- アウツ計算へ危険アウツを追加し、クリーンアウツを計算
- パワーナンバーへFirst in条件を追加
- ICM局面では参考値、First inでない場合は適用外と表示

### 精度上の注意

レンジの9-max、アンティなし、ICM、オープンサイズ差は学習用の簡易補正です。正確なGTO頻度を自動生成せず、自分のソルバー値を頻度編集で登録できる方式にしています。

## 完全勝率計算について

- モンテカルロ法や乱数は使用せず、残りカードの全組み合わせを列挙します。
- 相手のハンドを指定した場合は、プリフロップを含めて完全計算できます。
- 相手がランダムの場合は、フロップ・ターン・リバーから完全計算できます。
- プリフロップ対ランダムは2,097,572,400通りになるため、スマホでは相手の2枚を指定する仕様です。
- 重い計算はWeb Workerで実行し、進捗表示と計算中止に対応しています。

## GitHub Pagesで公開

1. 新しいGitHubリポジトリを作成
2. ZIPを解凍し、フォルダ内のファイルをすべてリポジトリ直下へアップロード
3. `Settings` → `Pages` を開く
4. Sourceを「Deploy from a branch」にする
5. Branchを `main`、Folderを `/ (root)` にして保存
6. 公開URLをスマホのChromeで開き、「ホーム画面に追加」

既にGitHub Pagesへ公開している場合は、ZIP内のファイルをすべて上書きしてください。
